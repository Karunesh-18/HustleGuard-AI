## Overview

Phase 1 focuses on extracting maximum value from the existing synthetic training data through systematic optimization without collecting new real-world data. This phase includes four core activities:

1. **Hyperparameter Tuning** – GridSearch + RandomSearch to optimize model parameters
2. **Feature Selection** – Identify and drop low-importance/redundant features
3. **Threshold Optimization** – Determine ideal decision threshold for disruption classification
4. **Cross-Validation** – Validate generalization and model stability

## Key Decisions During Phase 1

### 1. Hyperparameter Tuning Trade-offs

**What to tune**:
- `n_estimators`: [100, 250, 500] – more trees → better but slower
- `max_depth`: [10, 15, 20, None] – deeper trees → overfitting risk
- `min_samples_leaf`: [1, 2, 4] – smaller → more granular, more overfit-prone

**Our approach**: RandomizedSearchCV (40 iterations) instead of GridSearchCV (exhaustive) to save time while exploring space well.

### 2. Feature Dropping Strategy

**Minimum importance threshold**: 1% of total importance

**Rationale**:
- Features <1% contribute negligibly to predictions
- Removing them speeds up inference (<2% latency improvement)
- Lower complexity = easier interpretation, less overfitting

**Action**: Only drop if BOTH model importance AND permutation importance <1%

### 3. Threshold Selection

**Business context**:
- **High false positive cost** (over-compensating workers): Lower threshold to ~0.3-0.4 (higher recall)
- **High false negative cost** (missing real disruptions): Raise threshold to ~0.6-0.7 (higher precision)
- **Balanced** (default): Use F1-optimal threshold (~0.5)

**Our recommendation**: Start with F1-optimal from `threshold_analysis.json`; adjust based on cost analysis later.

## Monitoring & Validation During Phase 1

### Sanity Checks (run after each step):

1. **After Hyperparameter Tuning**:
   - CV scores should be reasonable (R² > 0.98 for regression)
   - Training metrics > CV metrics (no distribution mismatch)

2. **After Feature Selection**:
   - Retrain with recommended features
   - Test set performance should not drop >0.5%

3. **After Threshold Optimization**:
   - F1-optimal threshold should differ from default 0.5 if imbalanced
   - ROC-AUC should be >0.95

4. **After CV Training**:
   - CV std dev <2% indicates good generalization
   - Training metrics > CV metrics confirms no data leakage

## Troubleshooting

### Issue: "ModuleNotFoundError: No module named 'hyperparameter_tuning'"

**Fix**: Ensure you're running from `backend/ml/` directory:
```bash
cd c:\Users\karunesh\CODE\HustleGuard-AI\backend\ml
python run_phase1.py
```

### Issue: "Dataset file not found: ./datasets/training_data.csv"

**Fix**: Run dataset_generator.py first:
```bash
python dataset_generator.py
python train_models.py
python run_phase1.py
```

### Issue: Hyperparameter tuning is taking >1 hour

**Solution**: Reduce `n_iter` in `hyperparameter_tuning.py`:
- Change from `n_iter=40` to `n_iter=20`
- Or run sequentially instead of parallel: `n_jobs=1`
